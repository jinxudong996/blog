未来四周学习规划

#### 掌握目标

1. embedding原理

   不用推公式。

   你需要理解：

   - 向量空间是什么意思
   - 余弦相似度
   - 维度对语义表达的影响
   - 不同 embedding 模型差异

   比如：

   - 文本 embedding vs code embedding
   - 多语言 embedding

   你要能回答：

   > 为什么同一句话用不同 embedding 模型召回结果差异很大？

2. chunk策略

   你要深入：

   - 固定长度切分
   - 语义切分
   - 滑动窗口
   - 标题分层切分
   - 父子块结构（Parent-Child Retrieval）

   现实中：

   > 80% RAG 问题来自切分策略

3. 向量数据库

   - HNSW 原理
   - IVF
   - ANN（近似最近邻）
   - TopK 的 tradeoff
   - 召回率 vs 速度

   可以选一个主攻：

   - pgvector（简单）
   - 或 Milvus（专业）

   重点不是会命令，是理解：

   > 为什么 TopK=3 和 TopK=10 效果完全不同？

4. 检索优化，召回层

   你需要深入：

   ### 1️⃣ Query Rewrite

   - 问题扩写
   - 同义词扩展

   ### 2️⃣ Rerank

   第一次召回 → 再排序

   ### 3️⃣ 混合检索

   - BM25（关键词）
   - 向量检索
   - Hybrid

   ### 4️⃣ 多跳检索

   复杂问题拆解。

5. 生成优化，prompt层

   很多人以为：

   > 检索准了就行

   不对。

   你还要学：

   - Prompt 模板设计
   - 引用约束
   - 防幻觉策略
   - 上下文压缩

6. 评估体系 质量保证

   非常重要。

   你要学习：

   ### 1️⃣ Offline 评估

   - 精确率
   - 召回率
   - MRR

   ### 2️⃣ 在线评估

   - 人工打分
   - A/B 测试

   ### 3️⃣ LLM as Judge

   用模型评估答案。

你可以清楚回答：

- 为什么 chunk 不能太大？
- 为什么 embedding 维度影响召回？
- 为什么 rerank 能提升质量？
- 如何用指标评估 RAG？

如果你能回答这些，你已经超过 70% 的 RAG 初级工程师。
