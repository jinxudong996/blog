##### 数据类型

mysql是将数据存储在表里，而表是由行列组成的，每一行就是数据的详细信息，每一列就是基本信息的属性，而这个属性实际上是有格式的。接下来详细介绍下mysql中支持的数据类型有那些。

###### 数值类型

数值主要分为整数和浮点点数

先来看整数，根据设置不同的整数类型，所占据的存储空间也不一样，如下表：

| 类型                     | 占用的存储空间（单位：字节） | 无符号数取值范围 | 有符号数取值范围 | 含义           |
| ------------------------ | ---------------------------- | ---------------- | ---------------- | -------------- |
| `TINYINT`                | 1                            | 0 ~ 2⁸-1         | -2⁷ ~ 2⁷-1       | 非常小的整数   |
| `SMALLINT`               | 2                            | 0 ~ 2¹⁶-1        | -2¹⁵ ~ 2¹⁵-1     | 小的整数       |
| `MEDIUMINT`              | 3                            | 0 ~ 2²⁴-1        | -2²³ ~ 2²³-1     | 中等大小的整数 |
| `INT`（别名：`INTEGER`） | 4                            | 0 ~ 2³²-1        | -2³¹ ~ 2³¹-1     | 标准的整数     |
| `BIGINT`                 | 8                            | 0 ~ 2⁶⁴-1        | -2⁶³ ~ 2⁶³-1     | 大整数         |

浮点数同理，看下表格

| 类型     | 占用的存储空间（单位：字节） | 绝对值最小非0值          | 绝对值最大非0值          | 含义         |
| -------- | ---------------------------- | ------------------------ | ------------------------ | ------------ |
| `FLOAT`  | 4                            | ±1.175494351E-38         | ±3.402823466E+38         | 单精度浮点数 |
| `DOUBLE` | 8                            | ±2.2250738585072014E-308 | ±1.7976931348623157E+308 | 双精度浮点数 |

浮点数在计算器底层采用的都是是二进制存储，遵循IEEE754标准，这个和js一样，比如输入`0.1+0.2`都会出现异常情况，于是mysql又推出了定点数。

>  定点数是一种精确的小数，为了达到精确的目的我们就不能把它转换成二进制小数之后再存储(因为有很多十进制小数转为二进制小数后需要进行舍入操作，导致二进制小数表示的数值是不精确的)。其实转念一想，所谓的小数只是把两个十进制整数用小数点分割开来而已，我们只要把小数点左右的两个十进制整数给存储起来，那不就是精确的了么。比方说对于十进制小数`2.38`来说，我们可以把这个小数的小数点左右的两个整数，也就是`2`和`38`分别保存起来，那么不就相当于保存了一个精确的小数 

其占用的存储空间如下：

| 类型            | 占用的存储空间（单位：字节） | 取值范围   |
| --------------- | ---------------------------- | ---------- |
| `DECIMAL(M, D)` | 取决于M和D                   | 取决于M和D |



###### 日期和时间类型

时间格式主要又如下几种

| 类型        | 存储空间要求           | 取值范围                                                     | 含义         |
| ----------- | ---------------------- | ------------------------------------------------------------ | ------------ |
| `YEAR`      | 1字节                  | 1901~2155                                                    | 年份值       |
| `DATE`      | 3字节                  | '1000-01-01' ~ '9999-12-31'                                  | 日期值       |
| `TIME`      | 3字节+小数秒的存储空间 | '-838:59:59[.000000]' ~ '838:59:59[.000000]'                 | 时间值       |
| `DATETIME`  | 5字节+小数秒的存储空间 | '1000-01-01 00:00:00[.000000]' ～ '9999-12-31 23:59:59'[.999999] | 日期加时间值 |
| `TIMESTAMP` | 4字节+小数秒的存储空间 | '1970-01-01 00:00:01[.000000]' ～ '2038-01-19 03:14:07'[.999999] | 时间戳       |



###### 字符串类型

字符大致分为两种：可见字符和不可见字符，其中可见字符就是打印后可以看到的，不可见字符主要是一些换行符这种。

看如下表格：

| 类型         | 最大长度     | 存储空间要求      | 含义             |
| ------------ | ------------ | ----------------- | ---------------- |
| `CHAR(M)`    | M个字符      | M×W个字节         | 固定长度的字符串 |
| `VARCHAR(M)` | M个字符      | L+1 或 L+2 个字节 | 可变长度的字符串 |
| `TINYTEXT`   | 2⁸-1 个字节  | L+1个字节         | 非常小型的字符串 |
| `TEXT`       | 2¹⁶-1 个字节 | L+2 个字节        | 小型的字符串     |
| `MEDIUMTEXT` | 2²⁴-1 个字节 | L+3个字节         | 中等大小的字符串 |
| `LONGTEXT`   | 2³²-1 个字节 | L+4个字节         | 大型的字符串     |

展示表

创建表就麻烦点了，有这么几个规则

1. 给表起个名。
2. 给表定义一些列，并且给这些列都起个名。
3. 每一个列都需要定义一种数据类型。
4. 如果有需要的话，可以给这些列定义一些列的属性，比如不许存储`NULL`，设置默认值等等，具体列可以设置哪些属性我们稍后详细唠叨。

```
CREATE TABLE first_table (
    first_column INT,
    second_column VARCHAR(100)
);
```

修改表名可以通过两种方法：

1.  ALTER **TABLE** 旧表名 RENAME **TO** 新表名; 
2.  RENAME **TABLE** 旧表名1 **TO** 新表名1, 旧表名2 **TO** 新表名2, ... 旧表名n **TO** 新表名n; 

增加列：

```
ALTER TABLE 表名 ADD COLUMN 列名 数据类型 [列的属性];
```

增加列到指定位置：

```
ALTER TABLE 表名 ADD COLUMN 列名 列的类型 [列的属性] FIRST;
```

添加到指定列的后边：

```
ALTER TABLE 表名 ADD COLUMN 列名 列的类型 [列的属性] AFTER 指定列名;
```

删除列：

```
ALTER TABLE 表名 DROP COLUMN 列名;
```

修改列名：

```
ALTER TABLE 表名 MODIFY 列名 新数据类型 [新属性];
```

mysql插入数据是以行为单位的：

```
INSERT INTO 表名(列1, 列2, ...) VALUES(列1的值，列2的值, ...);
```



###### 列的属性

1. 默认值

   一般如果没有显示设置列的默认值，他都会被默认设置为`NULL`的，也可以理解为设置为`NULL`的都是还没有被设置的

   如果要设置默认值，再创建时可以显示指定

   ```
   CREATE TABLE first_table (
       first_column INT DEFAULT NULL,
       second_column VARCHAR(100) DEFAULT 'abc'
   );
   ```

   

2. NOT NULL

   如果我们要指定某一列必须有值，不能存放`NULL`

   ```
   列名 列的类型 NOT NULL
   ```

   这就意味着再使用`insert`插入语句时，必须要显示指定列的值。

3. 主键

   在表里，可以通过某些列或者某些列确定唯一的一条记录，我们就可以把这个列或者这些列称之为候选键。一个表可以有多个候选键，可以选择一个作为表的主键，一个表最多只能有一个主键，而且主键的值不能重复。

   定义主键的方式有两种：

   1.  如果主键只是单个列的话，可以直接在该列后声明`PRIMARY KEY`，比如我们把学生信息表`student_info`的`学号`列声明为主键可以这么写： 

      ```
      CREATE TABLE student_info (
          number INT PRIMARY KEY,
          name VARCHAR(5),
          sex ENUM('男', '女'),
          id_number CHAR(18),
          department VARCHAR(30),
          major VARCHAR(30),
          enrollment_time DATE
      );
      ```

   2.  我们也可以把主键的声明单独提取出来 

      ```
      PRIMARY KEY (列名1, 列名2, ...)
      ```

      比如这样：

      ```
      CREATE TABLE student_info (
          number INT,
          name VARCHAR(5),
          sex ENUM('男', '女'),
          id_number CHAR(18),
          department VARCHAR(30),
          major VARCHAR(30),
          enrollment_time DATE,
          PRIMARY KEY (number)
      );
      ```

      

4. UNIQUE属性

    对于不是主键的其他候选键，如果也想让`MySQL`在我们向表中插入新记录的时候帮助我们校验一下某个列或者列组合的值是否重复，那么我们可以把这个列或列组合添加一个`UNIQUE`属性，表明该列或者列组合的值是不允许重复的。 

   也有两种申明方式，和申明主键基本类似

   1.  我们想为单个列声明`UNIQUE`属性，可以直接在该列后填写`UNIQUE`或者`UNIQUE KEY` 
   2.  UNIQUE [约束名称] (列名1, 列名2, ...) 

   主键和UNIQUE都是保证列中的数据不重复，但是一张表中只能有一个主键，可以申明多个UNIQUE。

   

5. 外键

   用来保证两个表之间数据关系一致性的约束机制，定义了一个表中的某个字段必须引用另外一个表中的主键或唯一键。

   ```
   CONSTRAINT [外键名称] FOREIGN KEY(列1, 列2, ...) REFERENCES 父表名(父列1, 父列2, ...);
   ```

   

6. AUTO_INCREMENT属性

   这个就是列中的自增属性，常见的做法就是指定某一列，为其添加自增属性，比如id，后续插值时就可以不用管它，他的值会自动增长。

   

###### 简单查询

一般涉及到数据库的，最常见的需求就是各种查询，当然本次学习的重点也是查询。

1. 查询单列

   ```
   SELECT 列名 FROM 表名;
   ```

   

2. 查询多列

   ```
   SELECT 列名1, 列名2, ... 列名n FROM 表名;
   ```

   比如这样：

   ```
   mysql> SELECT number, name, id_number, major FROM student_info;
   +----------+------+--------------------+----------+
   | number   | name | id_number          | major    |
   +----------+------+--------------------+----------+
   | 20180101 | ???  | 158177199901044792 | ???????? |
   | 20180102 | ???  | 151008199801178529 | ???????? |
   | 20180103 | ??   | 17156319980116959X | ????     |
   | 20180104 | ???  | 141992199701078600 | ????     |
   | 20180105 | ??   | 181048199308156368 | ?????    |
   | 20180106 | ???  | 197995199501078445 | ????     |
   +----------+------+--------------------+----------+
   6 rows in set (0.00 sec)
   ```

   

3. 查询所有列

   ```
   SELECT * FROM 表名;
   ```

   

4. 查询结果去重

   查询时加个关键字即可

   ```
   SELECT DISTINCT 列名 FROM 表名;
   ```

   

5. 限制查询结果条数

   这里是在查询结尾加上关键字和查询位置

   ```
   LIMIT 开始行, 限制条数;
   ```

   

6. 查询结果排序

   如果要对查询的结果进行排序，可以这么做

   ```
   ORDER BY 列名 ASC|DESC
   ```

   比如：

   ```
    SELECT * FROM student_score ORDER BY score ASC;
   ```

   





###### 带搜索条件的查询

1. 简单搜搜条件

   可以通过where关键字配合一些搜索条件完成简单查询

   ```
   SELECT * FROM student_score WHERE score > 70;
   ```

   上面就是查询`student_score`表中的`score`大于70的数据

   操作符总结了下，有这几个

   | 操作符        | 示例                    | 描述               |
   | ------------- | ----------------------- | ------------------ |
   | `=`           | `a = b`                 | a等于b             |
   | `<>`或者`!=`  | `a <> b`                | a不等于b           |
   | `<`           | `a < b`                 | a小于b             |
   | `<=`          | `a <= b`                | a小于或等于b       |
   | `>`           | `a > b`                 | a大于b             |
   | `>=`          | `a >= b`                | a大于或等于b       |
   | `BETWEEN`     | `a BETWEEN b AND c`     | 满足 b <= a <= c   |
   | `NOT BETWEEN` | `a NOT BETWEEN b AND c` | 不满足 b <= a <= c |

2. 匹配表中元素

   当搜索条件不是匹配单个值，而是一个列表时，可以使用in操作符

   比如

   ```
   SELECT number, name, id_number, major FROM student_info WHERE major IN ('软件工程', '飞行器设计');
   ```

   

3. 匹配NULL

   当某一行没有值时，默认就是NULL，而判断某一行的值是不是NULL，不能使用`=`，需要专门的操作符

   | 操作符        | 示例            | 描述            |
   | ------------- | --------------- | --------------- |
   | `IS NULL`     | `a IS NULL`     | a的值是`NULL`   |
   | `IS NOT NULL` | `a IS NOT NULL` | a的值不是`NULL` |

   比如：

   ```
   SELECT number, name, id_number, major FROM student_info WHERE name IS NOT NULL;
   ```

   

4. 多个搜索条件

   1.  AND操作符

      理解也很简单，就是两个条件取交集

      ```
       SELECT * FROM student_score WHERE subject = '母猪的产后护理' AND score > 75;
      ```

      

   2. OR操作符

      这个操作符就是两个条件取并集

      ```
      SELECT * FROM student_score WHERE score > 95 OR score < 55;
      ```

      

5. 通配符

   有时候不能精准描述要查询的结果，就需要通配符来做模糊查询，需要下边两个操作符来支持

   | 操作符     | 示例           | 描述     |
   | ---------- | -------------- | -------- |
   | `LIKE`     | `a LIKE b`     | a匹配b   |
   | `NOT LIKE` | `a NOT LIKE b` | a不匹配b |

   操作符通常还要搭配通配符来使用

   1. % 代表任意字符

      ```
      SELECT number, name, id_number, major FROM student_info WHERE name LIKE '杜%';
      ```

      这就是查找杜姓为开头的数据

   2. _代表任意一个字符

      ```
      SELECT number, name, id_number, major FROM student_info WHERE name LIKE '范_';
      ```

      这里查找范x的数据，当然不包含范xx



###### 表达式和函数

1. 表达式

   就是在查询语句中，可以对列做一些数学运算，比如加减乘除取余等。

   比如

   ```
    SELECT  number, subject, score + 100 FROM student_score;
   ```

   这条查询就给` score `这一列加了一百

2. 函数

   mysql内置了很多工具函数，下面统计下

   | 名称        | 调用示例                      | 示例结果    | 描述                                   |
   | ----------- | ----------------------------- | ----------- | -------------------------------------- |
   | `LEFT`      | `LEFT('abc123', 3)`           | `abc`       | 给定字符串从左边取指定长度的子串       |
   | `RIGHT`     | `RIGHT('abc123', 3)`          | `123`       | 给定字符串从右边取指定长度的子串       |
   | `LENGTH`    | `LENGTH('abc')`               | `3`         | 给定字符串的长度                       |
   | `LOWER`     | `LOWER('ABC')`                | `abc`       | 给定字符串的小写格式                   |
   | `UPPER`     | `UPPER('abc')`                | `ABC`       | 给定字符串的大写格式                   |
   | `LTRIM`     | `LTRIM(' abc')`               | `abc`       | 给定字符串左边空格去除后的格式         |
   | `RTRIM`     | `RTRIM('abc ')`               | `abc`       | 给定字符串右边空格去除后的格式         |
   | `SUBSTRING` | `SUBSTRING('abc123', 2, 3)`   | `bc1`       | 给定字符串从指定位置截取指定长度的子串 |
   | `CONCAT`    | `CONCAT('abc', '123', 'xyz')` | `abc123xyz` | 将给定的各个字符串拼接成一个新字符串   |
